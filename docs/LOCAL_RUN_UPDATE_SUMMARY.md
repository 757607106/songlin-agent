## 📌 重要提示

### 开发者必读
1. **一定要先启动 Docker 基础服务** → 其他所有服务依赖它
2. **使用 3 个终端窗口** → 便于监控和调试
3. **保持日志可见** → 及时发现问题

### 常见误区
- ❌ 直接运行后端而不启动 Docker → 会连接失败
- ❌ 在同一终端运行多个服务 → 无法正常启动后续服务
- ❌ 忽视启动日志 → 无法判断服务是否就绪

### 快速参考
```bash
# 最快的启动方式
# 终端 1
make local-deps

# 终端 2
make local-api

# 终端 3
make local-web

# 浏览器访问
http://localhost:5173
```
